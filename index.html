当我们谈论Vue时，我们在谈论什么（一）

发表于 2019-03-06 | 分类于 Vue

从定位上来说，这个系列的文章并不打算从写一个 Hello World 开始，按照官方文档的顺序将 Vue 的所有特性全部介绍一遍。重复的轮子没多大意义，特别是当官方文档和资源都已经十分齐备的情况下。

那么，我们应该谈些什么呢？我想，围绕几个特定主题来谈论，是比较好的方法。通过这种方式，可以将官方文档重新组织一遍，如果顺利的话，我期望可以将 Vue 那些有趣又强大的特性以另一种方式呈现出来，作为官方文档的补充。

当然，这个目标很大，而笔者也并非 Vue 的资深开发者，所以我并不期望这个系列的文章可以作为一套入门教程。相反的，我期望它们可以作为一些话题，一些关于 Vue 的话题。就像是冬天围坐在火炉边的人们一样，我希望可以在这里忘记其他事情，而去感受 Vue 这个框架中那些令人激动的特性。

事实上从使用者的角度来说，Vue 仅有视图和逻辑两个层面。换句话说，我们仅仅需要关系数据将如何展示，以及这些数据将被如何驱动。尽管这并不能包含所有的必需特性，但从开发的角度来说，这的确是接触最高频率的功能需求。

首先，我会谈一谈关于视图和数据。
其次，我将会谈谈我们将如何组织这些文件，它们之间将如何建立联系。事实上你将会发现，通过引入模块化的思维，这将是很容易被理解的
第三，我将会谈一谈 Vue 的生态，也就是那些被称为 Vue 全家桶的东西。它们究竟是一些什么？它们究竟应该被用在什么地方？
第四，我将会谈谈那些可以加速你开发或者可以帮助你更好的组织文件的插件或是工具，抑或是一些开发技巧。
最后，如果可能的话，我希望可以就 Vue 的一些核心功能的源码做讲解。
